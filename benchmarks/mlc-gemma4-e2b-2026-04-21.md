3 runs, 256-token generation limit, `temperature=0.0` (greedy), prompt: "Write a short poem about the ocean."

## Compile Command

```bash
sudo docker run --rm --runtime nvidia \
  -v /opt/models/gemma4-e2b-q4f16-mlc:/weights \
  rimrock/mlc-gemma4:latest \
  python3 -m mlc_llm compile /weights \
    --device cuda \
    --opt O3 \
    --output /weights/lib-sm87.so
```

Output: `/opt/models/gemma4-e2b-q4f16-mlc/lib-sm87.so` (38MB)

## Notes

- FlashInfer `batch_prefill` JIT op fails to load (missing `.sources` attr) — model falls back to TIR-based KV cache. This is a known limitation of the PR branch.
- CUTLASS and cudagraph enabled via O3 opt preset.
- `mlc_llm bench` subcommand not present in this PR branch; benchmark used `MLCEngine` Python API directly.
